    Introduction to Strings

    What is a String?

    Definition

    A String in Java is a sequence of characters that represents text data. It is one of the most commonly used data types in Java programming and is used to store and manipulate textual information.

    Characteristics of Strings

    1. Immutable: Once a String object is created, its value cannot be changed
    2. Object Type: String is a class, not a primitive data type
    3. Unicode Support: Java Strings use UTF-16 encoding, supporting international characters
    4. Sequence of Characters: Internally, a String is backed by a character array

    String Representation

    // Simple string representation
String message = "Hello, World!";
String name = "John Doe";
String empty = "";

    Internal Structure

    Internally, a String object contains:

    • A character array (char[]) or byte array (byte[]) in Java 9+
    • A hash code value (cached for performance)
    • Length information

    String class declaration 

    // String class declaration (simplified)
public final class String
    implements java.io.Serializable, Comparable<String>, CharSequence {

    // Java 9+: byte array for compact storage
    private final byte[] value;

    // Java 8 and below: char array
    // private final char value[];
}

    Example: 

    String text = "Java";
// Internally represented as:
// char[] value = {'J', 'a', 'v', 'a'};
// int hash = 0; (initially)

    String as Text Container

    Strings can contain:

    • Letters: "ABC", "xyz"
    • Numbers: "12345", "3.14" (as text, not numeric values)
    • Special characters: "@#$%", "Hello\nWorld" (with escape sequences)
    • Unicode characters: "Hello 世界", "Café"
    • Empty strings: ""
    • Whitespace: " ", "\t\n"

    Examples: 

    String letters = "Programming";
String numbers = "2024";
String mixed = "Java 17";
String special = "Price: $99.99";
String unicode = "Namaste नमस्ते";
String multiline = "Line 1\nLine 2\nLine 3";

    Common Use Cases

    1. User Input/Output 

      String username = "alice123";
String password = "secure@Pass";

    2. File Operations 

      String filename = "data.txt";
String filepath = "/home/user/documents/";

    3. Data Representation 

      String jsonData = "{\"name\":\"John\",\"age\":30}";
String xmlData = "<user><n>John</n></user>";

    4. Messages and Labels 

      String errorMessage = "Invalid input!";
String successMessage = "Operation completed successfully";

    String as a Class vs Primitive Types

    Understanding the Difference

    Java has two categories of data types:

    1. Primitive Types: byte, short, int, long, float, double, char, boolean
    2. Reference Types (Objects): All classes, including String

    String is a Class

    String is defined as a class in Java:

    public final class String
    implements java.io.Serializable, Comparable<String>, CharSequence {
    // Implementation details
}

    Key Point: String is NOT a primitive type; it's a full-fledged class with methods and properties.

    Comparison: Primitive vs String

    Aspect Primitive Types String (Class)
    Memory Location Stack Heap (object) + Stack (reference)
    Default Value 0, false, '\u0000' null
    Methods No methods Many methods (length(), charAt(), etc.)
    Mutability Mutable (can change value) Immutable (cannot change)
    Performance Faster Slightly slower (object overhead)
    Size Fixed size Variable size
    Comparison Use == Use equals() method

    Primitive Type Example

    // Primitive type
int number = 42;
number = 100; // Value changed directly in memory

// Primitive comparison
int a = 10;
int b = 10;
System.out.println(a == b); // true (compares values)

    String (Class) Example

    // String object
String text = "Hello";
text = "World"; // Creates new object, old object remains in memory

// String comparison
String s1 = "Hello";
String s2 = "Hello";
System.out.println(s1 == s2);        // true (same object due to string pool)
System.out.println(s1.equals(s2));   // true (compares content)

String s3 = new String("Hello");
String s4 = new String("Hello");
System.out.println(s3 == s4);        // false (different objects)
System.out.println(s3.equals(s4));   // true (same content)

    Why String is Special Among Classes

    Despite being a class, String has some special privileges:

    1. Literal Support: Unlike other classes, String can be created using literals 

      String s1 = "Hello";              // Literal (like primitives)
Integer i1 = new Integer(10);     // No literal support for Integer
// Note: Integer i1 = 10; uses autoboxing, not true literal

    2. Operator Overloading: String supports the + operator for concatenation 

      String result = "Hello" + " " + "World"; // Operator overloading
// No other class in Java supports operator overloading

    3. Constant Pool: String has a special memory area called String Pool 

      String s1 = "Java";  // Stored in String Pool
String s2 = "Java";  // References same object in pool

    Memory Layout

    Primitive Variable: 

    Stack Memory:
+----------------+
| number = 42    |
+----------------+

    String Variable: 

    Stack Memory:              Heap Memory:
+------------------+      +------------------------+
| text (reference) |----> | String object "Hello"  |
+------------------+      | - value: char[]        |
                          | - hash: int            |
                          +------------------------+

    Autoboxing vs String Literal

    // Primitive wrapper (autoboxing)
Integer num = 100;  // Autoboxed from int to Integer

// String literal
String text = "Hello";  // True literal support, not boxing

// The difference:
Integer i = 100;  // Actually: Integer.valueOf(100)
String s = "Hello"; // Direct literal, stored in pool

    Code Example: Class Behavior

    public class StringAsClass {
    public static void main(String[] args) {
        // String has methods like any class
        String text = "Java Programming";

        // Method calls (proves it's a class)
        int length = text.length();              // 16
        String upper = text.toUpperCase();       // "JAVA PROGRAMMING"
        boolean starts = text.startsWith("Java"); // true
        char ch = text.charAt(0);                // 'J'

        // String is an object
        System.out.println(text.getClass());     // class java.lang.String
        System.out.println(text instanceof String); // true
        System.out.println(text instanceof Object); // true

        // Has Object class methods
        String copy = text.toString();
        int hashCode = text.hashCode();
        boolean equals = text.equals("Java Programming");
    }
}

    null vs Empty String

    Because String is a class:

    // null - no object exists
String s1 = null;  // Valid for objects, not primitives
// int x = null;   // Compilation error!

// Empty string - object exists, contains no characters
String s2 = "";

// Comparison
System.out.println(s1 == null);     // true
System.out.println(s2 == null);     // false
System.out.println(s2.length());    // 0
// System.out.println(s1.length()); // NullPointerException!

    Why Strings are Special in Java

    Special Features of String

    String has unique treatment in Java that makes it different from other classes. Here are the key reasons why Strings are special:

    1. String Literals and String Pool

    Special Feature: Strings can be created using literals, and Java maintains a special memory region called the String Pool.

    // String literal - stored in String Pool
String s1 = "Hello";
String s2 = "Hello";
System.out.println(s1 == s2); // true - same object in pool

// Using new keyword - creates object in heap
String s3 = new String("Hello");
String s4 = new String("Hello");
System.out.println(s3 == s4); // false - different objects

    Why This Matters:

    • Memory optimization: Identical string literals share the same object
    • Performance improvement: No need to create duplicate strings
    • Automatic interning: JVM automatically interns string literals

    2. Immutability

    Special Feature: String objects are immutable - once created, their value cannot be changed.

    String original = "Java";
String modified = original.concat(" Programming");

System.out.println(original);  // "Java" - unchanged
System.out.println(modified);  // "Java Programming" - new object

    Benefits of Immutability:

    1. Security: String is used for sensitive data (passwords, URLs, file paths) 

      String password = "secret123";
// No method can modify the original password
String upper = password.toUpperCase(); // Creates new String
// password is still "secret123"

    2. Thread Safety: Multiple threads can share String objects safely 

      public class ThreadSafeExample {
    private static String sharedString = "Shared Data";

    // Multiple threads can read without synchronization
    public void readString() {
        System.out.println(sharedString);
    }
}

    3. Caching Hash Code: Once calculated, hash code never changes 

      String key = "username";
int hash1 = key.hashCode();
// Can cache this value - it will never change
int hash2 = key.hashCode();
// Same hash value guaranteed

    4. String Pool Optimization: Safe to share string literals 

      String s1 = "constant";
String s2 = "constant";
// Safe to share same object - it cannot be modified

    3. Operator Overloading for + (Concatenation)

    Special Feature: String is the ONLY class in Java that supports operator overloading.

    // + operator for concatenation
String result = "Hello" + " " + "World"; // "Hello World"

// Works with other types (auto-conversion)
String message = "Count: " + 42;         // "Count: 42"
String price = "Price: $" + 99.99;       // "Price: $99.99"

// Behind the scenes (Java compiler optimization)
String s = "Hello" + " " + "World";
// Compiled to:
// String s = new StringBuilder().append("Hello").append(" ").append("World").toString();

    No Other Class Has This: 

    Integer a = 10;
Integer b = 20;
Integer c = a + b;  // Unboxing to int, then addition, then boxing
// NOT object method call

ArrayList<String> list1 = new ArrayList<>();
ArrayList<String> list2 = new ArrayList<>();
// ArrayList<String> combined = list1 + list2; // Compilation error!

    4. Final Class (Cannot be Subclassed)

    Special Feature: String class is declared as final, preventing inheritance.

    public final class String { ... }

// This is not allowed:
// class MyString extends String { } // Compilation error!

    Why This Matters:

    • Security: Prevents malicious code from creating fake String subclasses
    • Consistency: All String objects behave exactly the same
    • Optimization: JVM can make aggressive optimizations

    Example of Security Importance: 

    // Imagine if String could be extended:
// class HackedString extends String {
//     public boolean equals(Object obj) {
//         return true; // Always return true!
//     }
// }
// This could break security checks, HashMap, etc.

    5. CharSequence Interface Implementation

    Special Feature: String implements CharSequence, making it compatible with other text types.

    // String implements CharSequence
CharSequence cs1 = "Hello";          // String
CharSequence cs2 = new StringBuilder("World"); // StringBuilder
CharSequence cs3 = new StringBuffer("Java");   // StringBuffer

// Methods accept CharSequence
public void printText(CharSequence text) {
    System.out.println(text);
}

printText("String literal");           // Works
printText(new StringBuilder("Builder")); // Works
printText(new StringBuffer("Buffer"));   // Works

    6. Automatic Type Conversion

    Special Feature: String automatically converts other types when using + operator.

    String s1 = "Number: " + 42;           // int to String
String s2 = "Pi: " + 3.14159;          // double to String
String s3 = "Boolean: " + true;        // boolean to String
String s4 = "Character: " + 'A';       // char to String

Object obj = new Object();
String s5 = "Object: " + obj;          // Object.toString() called

String s6 = "Null: " + null;           // "Null: null"

    7. Compile-Time Constant Folding

    Special Feature: String literals are evaluated at compile time.

    // Evaluated at compile time
String s1 = "Hello" + " " + "World";
// Compiler creates single string: "Hello World"

final String HELLO = "Hello";
final String WORLD = "World";
String s2 = HELLO + " " + WORLD;
// Also compile-time constant (because finals)

// Runtime concatenation
String hello = "Hello";  // Not final
String world = "World";
String s3 = hello + " " + world;
// Compiled to StringBuilder operations

    8. String Interning

    Special Feature: intern() method allows manual string pooling.

    String s1 = new String("Hello");
String s2 = new String("Hello");

System.out.println(s1 == s2);           // false (different objects)

String s3 = s1.intern();
String s4 = s2.intern();

System.out.println(s3 == s4);           // true (same pooled object)
System.out.println(s3 == "Hello");      // true (pooled literal)

    9. Rich API with Many Utility Methods

    Special Feature: String class provides extensive methods for text manipulation.

    String text = "Java Programming";

// Inspection methods
int length = text.length();              // 16
boolean empty = text.isEmpty();          // false
char ch = text.charAt(0);               // 'J'

// Comparison methods
boolean equals = text.equals("Java Programming");
boolean equalsIgnore = text.equalsIgnoreCase("java programming");
int comparison = text.compareTo("Python");

// Search methods
boolean contains = text.contains("Program");
int index = text.indexOf("Program");
boolean starts = text.startsWith("Java");
boolean ends = text.endsWith("ing");

// Modification methods (return new String)
String upper = text.toUpperCase();
String lower = text.toLowerCase();
String trimmed = "  text  ".trim();
String replaced = text.replace("Java", "Python");

// Extraction methods
String sub = text.substring(5);
String[] words = text.split(" ");
char[] chars = text.toCharArray();

    10. Special JVM Optimization

    Special Feature: JVM has special optimizations for String.

    1. String Deduplication (Java 8u20+) 

      // JVM can identify duplicate strings and deduplicate them
// -XX:+UseStringDeduplication

    2. Compact Strings (Java 9+) 

      // Strings with only Latin-1 characters use byte[] instead of char[]
String ascii = "Hello";  // Uses byte[] (1 byte per character)
String unicode = "Hello 世界"; // Uses char[] (2 bytes per character)

    Complete Example: Why Strings are Special

    public class StringSpecialFeatures {
    public static void main(String[] args) {
        // 1. Literal support + String Pool
        String s1 = "Java";
        String s2 = "Java";
        System.out.println("Same object: " + (s1 == s2)); // true

        // 2. Immutability
        String original = "Hello";
        original.concat(" World");
        System.out.println("Unchanged: " + original); // "Hello"

        // 3. Operator overloading
        String result = "Count: " + 42 + ", Value: " + 3.14;
        System.out.println(result);

        // 4. Cannot extend String (final class)
        System.out.println("Is final: " + 
            java.lang.reflect.Modifier.isFinal(String.class.getModifiers()));

        // 5. CharSequence interface
        CharSequence cs = "Text";
        System.out.println("Length: " + cs.length());

        // 6. Rich API
        String text = "Java Programming Language";
        System.out.println("Uppercase: " + text.toUpperCase());
        System.out.println("Contains 'Java': " + text.contains("Java"));
        System.out.println("Substring: " + text.substring(0, 4));

        // 7. String interning
        String s3 = new String("Interned").intern();
        String s4 = "Interned";
        System.out.println("Interned same: " + (s3 == s4)); // true

        // 8. Compile-time constant
        final String PART1 = "Com";
        final String PART2 = "piled";
        String compiled = PART1 + PART2;
        String literal = "Compiled";
        System.out.println("Compile-time: " + (compiled == literal)); // true
    }
}

    String Package Location (java.lang)

    Package Structure

    The String class is located in the java.lang package:

    package java.lang;

public final class String
    implements java.io.Serializable, 
               Comparable<String>, 
               CharSequence {
    // Implementation
}

    Full Qualified Name: java.lang.String

    The java.lang Package

    Definition: java.lang is the most fundamental package in Java, containing classes essential to the Java language.

    Why java.lang is Special

    1. Automatically Imported: No need to explicitly import
    // No import needed
public class Example {
    public static void main(String[] args) {
        String text = "Hello";     // java.lang.String
        Integer num = 100;         // java.lang.Integer
        System.out.println(text);  // java.lang.System
        Math.sqrt(16);             // java.lang.Math
    }
}

// If String was in another package:
// import com.custom.String; // Would be required
    1. Contains Core Classes: All fundamental classes are here

    Key Classes in java.lang Package

    // Object - root of class hierarchy
Object obj = new Object();

// String - text manipulation
String str = "text";

// Wrapper classes
Integer i = 100;
Double d = 3.14;
Boolean b = true;
Character c = 'A';
Long l = 1000L;
Float f = 2.5f;
Byte by = 10;
Short s = 100;

// System - standard I/O, utilities
System.out.println("Hello");
System.currentTimeMillis();

// Math - mathematical operations
Math.sqrt(25);
Math.max(10, 20);

// Thread - concurrency
Thread t = new Thread();

// Exception classes
Exception e = new Exception();
RuntimeException re = new RuntimeException();
NullPointerException npe = new NullPointerException();

// StringBuilder, StringBuffer - mutable strings
StringBuilder sb = new StringBuilder();
StringBuffer sbuf = new StringBuffer();

// Class - reflection
Class<?> cls = String.class;

    No Import Statement Required

    // These all work without import
public class NoImportNeeded {
    public static void main(String[] args) {
        // All from java.lang package
        String message = "Hello, World!";
        Integer number = Integer.parseInt("42");
        Double pi = Math.PI;
        System.out.println(message);
        Object obj = new Object();
        Thread thread = Thread.currentThread();
        Class<?> stringClass = String.class;
    }
}

    Accessing String Class

    // Simple usage (no qualification needed)
String s1 = "Hello";

// Fully qualified name (rarely needed)
java.lang.String s2 = "World";

// Through reflection
Class<?> stringClass = java.lang.String.class;

// Getting package information
Package pkg = String.class.getPackage();
System.out.println("Package: " + pkg.getName()); // java.lang

    String Class Location in JDK

    Source Location: In JDK source code, String is located at: 

    JAVA_HOME/lib/src.zip/java.base/java/lang/String.java

    Module: In Java 9+, String is part of the java.base module 

    // Module info
Module module = String.class.getModule();
System.out.println(module.getName()); // java.base

    Compile-Time vs Runtime

    Compile Time: Compiler recognizes java.lang classes automatically

    // Compiler knows String is java.lang.String
String text = "compile time";

    Runtime: JVM loads String class from rt.jar (Java 8) or modules (Java 9+)

    Java 8 and earlier:
  JAVA_HOME/jre/lib/rt.jar -> java/lang/String.class

Java 9 and later:
  JAVA_HOME/lib/modules -> java.base module -> java/lang/String.class

    Package Hierarchy

    java.lang (Core language support)
  ├── String
  ├── Object
  ├── System
  ├── Math
  ├── Thread
  ├── Class
  ├── Integer, Double, Boolean, etc.
  └── Exception, RuntimeException, etc.

java.lang.annotation (Annotations)
  ├── Annotation
  └── ElementType

java.lang.invoke (Method handles)
  └── MethodHandle

java.lang.ref (References)
  └── WeakReference

java.lang.reflect (Reflection)
  ├── Method
  ├── Field
  └── Constructor

    Why String is in java.lang

    Reasons:

    1. Fundamental Type: String is essential to almost every Java program
    2. Language Integration: Special language support (literals, + operator)
    3. Universal Usage: Used in standard I/O, exceptions, reflection, etc.
    4. Convenience: Automatic import means no boilerplate
    5. Core Functionality: Part of language core, not a utility

    Example: Package Verification

    public class StringPackageInfo {
    public static void main(String[] args) {
        // Get String class information
        Class<?> stringClass = String.class;

        // Package information
        Package pkg = stringClass.getPackage();
        System.out.println("Package Name: " + pkg.getName());
        // Output: java.lang

        System.out.println("Package Spec Version: " + pkg.getSpecificationVersion());
        System.out.println("Package Impl Version: " + pkg.getImplementationVersion());

        // Module information (Java 9+)
        Module module = stringClass.getModule();
        System.out.println("Module Name: " + module.getName());
        // Output: java.base

        System.out.println("Is Exported: " + module.isExported("java.lang"));
        // Output: true

        // Class full name
        System.out.println("Canonical Name: " + stringClass.getCanonicalName());
        // Output: java.lang.String

        // Simple name
        System.out.println("Simple Name: " + stringClass.getSimpleName());
        // Output: String
    }
}

    Summary

    Key Takeaways

    1. What is a String?

      • Sequence of characters representing text
      • Immutable object stored in heap memory
      • Supports Unicode (UTF-16) for international text

    2. String as a Class vs Primitive

      • String is a class, not a primitive type
      • Has methods and behaviors like other objects
      • Special literal support makes it look like a primitive
      • Compared using equals(), not ==

    3. Why Strings are Special

      • String Pool for memory optimization
      • Immutable for security and thread safety
      • Only class supporting + operator overloading
      • Final class preventing inheritance
      • Rich API with extensive methods
      • Special JVM optimizations

    4. Package Location

      • Located in java.lang package
      • Automatically imported (no import statement)
      • Part of core language functionality
      • In java.base module (Java 9+)

    Quick Reference

    // String creation
String s1 = "literal";              // String pool
String s2 = new String("object");   // Heap

// String is a class
s1.length();                        // Method call
s1.toUpperCase();                   // Returns new String
